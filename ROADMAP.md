# SelfMind 演进路线图

> 核心理念：**可视化 → 管理 → 服务**

---

## 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      SelfMind                           │
├─────────────┬─────────────────────┬─────────────────────┤
│   v1.0      │       v2.0          │       v3.0          │
│  可视化深化  │      记忆管理       │     服务化输出      │
├─────────────┼─────────────────────┼─────────────────────┤
│   toC       │       toB          │      to开发者        │
│  (个人用户)  │    (团队/部门)      │   (Agent集成)        │
└─────────────┴─────────────────────┴─────────────────────┘
```

---

## v1.0 可视化深化

**目标**：让人直观"看到"记忆的全貌

### 核心功能

| 模块 | 功能 | 状态 |
|------|------|------|
| 记忆图谱 | 节点/边关系可视化 | 🔄 边关系修复中 |
| 知识图谱 | 结构化知识网络 | 🔄 边为0 |
| 记忆健康 | 遗忘曲线、衰减预警 | 🔄 未激活 |
| 时间轴 | 记忆时间线播放 | ✅ 基础完成 |
| 主题切换 | 浅色/深色主题 | 🔄 优化中 |

### 技术任务

- [ ] 修复边关系逻辑（当前0条边）
- [ ] 激活遗忘引擎访问统计
- [ ] 完善记忆健康可视化
- [ ] 浅色主题重写
- [ ] 节点交互优化（拖拽、缩放）

### 交付标准

- 记忆图谱节点数 ≥ 180，边数 ≥ 500
- 遗忘曲线可展示
- 支持浅色/深色切换

---

## v2.0 记忆管理

**目标**：让人能够"管"记忆

### 核心功能

| 模块 | 功能 | 状态 |
|------|------|------|
| 记忆增删 | 创建、编辑、删除记忆 | ❌ 未实现 |
| 分类管理 | 自定义分类、标签体系 | 🔄 基础 |
| 导入导出 | JSON/MD 格式导入导出 | ❌ 未实现 |
| 批量操作 | 批量编辑、删除、归类 | ❌ 未实现 |
| 高级搜索 | 语义搜索、多条件过滤 | 🔄 简单 |

### 技术任务

- [ ] 记忆 CRUD API
- [ ] 分类/标签管理界面
- [ ] 导入导出功能（JSON/MD）
- [ ] 批量选择与操作
- [ ] 富文本编辑器
- [ ] 记忆关联推荐

### 交付标准

- 支持记忆的完整生命周期管理
- 支持批量导入导出
- 搜索响应时间 < 200ms

## v3.0 服务化输出

**目标**：让其他系统能够"用"记忆

### 核心功能

| 模块 | 功能 | 状态 |
|------|------|------|
| REST API | 记忆读写 API | ❌ 未实现 |
| Agent 集成 | 与 Hermes/Openclaw 对接 | 🔄 Hermes 读取 |
| 多数据源 | 记忆源统一管理 | 🔄 仅 Hermes |
| 插件系统 | 扩展点设计 | ❌ 未实现 |
| Webhook | 记忆变更通知 | ❌ 未实现 |

### 技术任务

- [ ] RESTful API 设计
- [ ] OpenAPI 文档
- [ ] 多数据源适配器（Hermes/Openclaw/Honcho）
- [ ] 记忆变更事件机制
- [ ] 认证与权限控制
- [ ] 插件 SDK 设计

### 交付标准

- 提供完整的 API 文档
- 支持 3+ 数据源
- 可嵌入其他 Agent 系统

## 版本依赖

```
v1.0 (可视化)
    │
    ├── 边关系修复 → v2.0 记忆管理（需要可视化选择节点）
    │
    └── API 基础   → v3.0 服务化（需要 CRUD API）
    
v2.0 (管理)
    │
    └── 分类体系   → v3.0 多数据源（统一分类标准）
    
v3.0 (服务)
    │
    └── 插件系统   → 未来生态扩展
```
