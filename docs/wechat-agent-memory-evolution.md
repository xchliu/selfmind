# 从记忆看Agent进化

## 一、现状：Agent的记忆困境

每个AI Agent每次启动，几乎都是一张白纸。

它不记得上次你纠正了什么，不记得这个项目拒绝做什么，不记得哪个方案已经试过失败了，不记得哪条路已经浪费了你一个下午。

这个问题不是没人关注——相反，几乎所有主流Agent框架都在尝试解决。但各自的方案，都停留在"存储"层面，没有触及"进化"。

## 二、各Agent的记忆管理方式

### OpenAI Codex CLI：项目级上下文，无个人记忆

Codex CLI的记忆机制是AGENTS.md——一个放在项目根目录的文件，告诉Agent这个项目的编码规范、技术栈、架构约定。

但它有几个根本缺陷：

**只记住项目，不记住人。** AGENTS.md是静态的项目配置，不是动态的个人记忆。换个项目，之前的经验全部丢失。你在项目A踩过的坑，不会自动提醒你在项目B注意。

**没有演变。** AGENTS.md是手动维护的文本文件，内容从写入到删除之间没有任何版本记录、强度变化、使用频率追踪。它只是一个"此刻的说明书"，不是"过程的记录仪"。

**没有衰减。** 旧的经验和新的规范混在一起，没有机制区分哪些还在用、哪些已经过时。文件越长，噪声越多，Agent反而更容易被无关信息干扰。

### Claude Code：类似思路，同样局限

Claude Code用CLAUDE.md做同样的事——项目级指令文件。它支持MCP协议接入外部记忆工具（比如mnemo），但自身没有内置的记忆生命周期管理。

这意味着Agent的记忆要么是"写在文件里的静态配置"，要么是"每次需要时去外部查一下"。前者不会演变，后者不会内化。

### Hermes：有记忆，但缺过程

Hermes是目前做得最深的。它的记忆体系有三层：

- **soul.md** — 高频内化层，Agent每次启动自动加载，相当于"生来就知道的东西"
- **MEMORY.md / USER.md** — 持久记忆文件，记录偏好、经验、纠错
- **Honcho Provider** — 低频参考层，通过API从历史对话中拉取相关上下文

Hermes比Codex和Claude Code多了一个关键维度——**持久性**。Agent不是每次从零开始，它带着历史经验进入新对话。

但Hermes的记忆体系仍然缺一层——**过程态**。

MEMORY.md是一个扁平的文本文件。你可以在里面加一条记忆，也可以删一条，但你不知道：

- 这条记忆是什么时候产生的？
- 它被修改了几次？每次改了什么？
- 它多久没被用了？强度在衰减吗？
- 它和其他记忆有什么关联？

MEMORY.md记录的是"此刻的状态"，不是"演变的过程"。就像一本只写结论不写推导的笔记——你看到结果，但不知道结果是怎么来的。

### mnemo：记忆生命周期，但仍是外部查询

mnemo是最近出现的一个有趣项目。它做了其他Agent框架没做的事：

- 记忆有生命周期（active → stale → superseded → archived）
- 反馈机制（Agent标记记忆是否有用、误导或过时）
- 写入守卫（自动检测重复、矛盾、低质量记忆）
- 时间线API（可以回溯记忆变化）

这些机制让mnemo比Codex和Claude Code更接近"真正的记忆管理"。

但mnemo的核心定位是**MCP Server**——Agent每次需要记忆时，通过MCP协议调用mnemo的查询工具。这意味着：

**记忆始终是外部的。** Agent不是"记得"，而是"去查了"。就像一个人每次做决策前都要翻日记本——有用，但不自然。

**记忆不会内化。** mnemo的愿景是未来通过LoRA/Adapter把高频记忆训练进模型行为，但这是路线图，不是现实。当前所有记忆都停留在"外部检索"层面。

**过程没有可视化。** mnemo有生命周期和时间线，但没有把"记忆如何演变"这件事本身做成可视化。你能看到一条记忆的状态是stale，但看不到整条演变轨迹——产生、强化、衰减、更新、归档——的完整故事。

## 三、人类记忆的启示

要真正解决Agent的记忆问题，我们需要回到一个更根本的参照系——人类的记忆。

### 记忆是分层的

人类记忆不是一块硬盘，而是多层次的活系统：

- **短期记忆** — 此刻正在处理的信息，几秒到几分钟就消失
- **长期记忆** — 经反复强化后稳定保留的知识和经验
- **程序性记忆** — 不需要 conscious recall 的技能和习惯，比如骑自行车
- **情景记忆** — 特定事件的完整经历
- **语义记忆** — 抽象的概念和事实

这些层次不是独立的仓库，而是互相转化的动态系统。短期记忆反复出现会巩固为长期记忆；长期记忆长期不用会自然衰减；程序性记忆不需要每次"想起"就能直接指导行为。

### 记忆是演变的

人类记忆不是写入即固定的。一条记忆从产生到稳定，经历了一个完整的演变过程：

**编码** — 事件发生，大脑将信息编码为记忆痕迹
**巩固** — 反复回忆或睡眠中的重播，让记忆从脆弱变得稳定
**强化** — 高频使用的记忆越来越强，提取越来越快
**衰减** — 长期不用的记忆逐渐模糊，最终被遗忘
**更新** — 新的信息会修改旧记忆，而不是简单覆盖

这个演变过程才是记忆的本质。一条记忆的"当前状态"只是演变过程中的一个切片，切片本身不能说明这条记忆是否健康、是否可靠、是否该被信任。

### 遗忘是健康的

人类大脑不是无限存储的仓库。遗忘不是故障，而是功能。

艾宾浩斯遗忘曲线告诉我们：不重要的信息会快速衰减，重要的信息会因为反复使用而保留。这种选择机制让大脑始终保持高效——有用的知识随时可用，过时的经验自动清理，为新知识腾出空间。

Agent的记忆系统也需要这种"健康遗忘"。不是所有记忆都应该永远保留——过时的技术决策、已经修正的错误认知、不再相关的工作流程，都应该有衰减和归档的机制。

### 记忆塑造行为

最关键的一点：人类记忆的最高形式不是"能想起来"，而是"已经影响了行为"。

你不需要每次骑车前翻笔记回忆怎么踩踏板——程序性记忆已经内化到了肌肉里。你不需要每次社交前查备忘录回忆这个人偏好什么——经验已经塑造了你的直觉反应。

这就是记忆进化的终极形态：**从"需要查询"到"已经成为行为的一部分"**。

## 四、SelfMind：Agent的DNA测序仪

回到Agent的世界。前面分析的所有方案，都在解决"记忆怎么存储和查询"的问题。但人类记忆启示我们，真正要解决的是"记忆怎么演变和进化"的问题。

SelfMind的定位不是记忆仓库，不是查询工具，不是MCP Server——它是**Agent的DNA测序仪**。

### 什么是Agent的DNA？

每个Agent在持续工作的过程中，经历的事情、踩过的坑、总结出的经验，都会沉淀为独特的记忆模式和行为特征。这些模式不是写死在代码里的，而是**用出来的**。

就像人类的DNA不是某一天被设计出来的，而是在亿万年的进化中，通过与环境交互、自然选择、基因突变，逐渐形成了当前的序列。Agent的DNA也一样——通过持续使用、记忆积累、经验提炼，逐渐形成独特的"记忆基因组合"。

Hermes的DNA包含：偏好简洁沟通、擅长项目管理、知道坦哥是AI部门负责人、了解商业银行的AI五年规划。这些不是天生写在soul.md里的全部——很多是在使用过程中积累和强化的。

不同Agent使用久了会形成不同的DNA：一个做数据分析的Agent擅长模式识别，一个做文档的Agent擅长结构化表达，一个做故障排查的Agent擅长快速定位根因。

### SelfMind记录什么？

SelfMind不记录Agent"此刻有什么记忆"——那是实时态，Hermes的soul.md和MEMORY.md已经在做。

SelfMind记录的是"记忆怎么变的"——这是过程态：

| 字段 | 含义 | 说明 |
|---|---|---|
| first_seen_at | 产生时间 | 这条记忆首次出现的时间 |
| version | 版本号 | 这条记忆被修改了几次 |
| updated_at | 更新时间 | 最近一次变化的时间 |
| decay_score | 记忆强度 | 高频使用则高，长期未用则衰减 |

这四个字段构成了Agent DNA的"碱基对"——每条记忆的演变轨迹都被忠实记录。

### DNA双螺旋模型

SelfMind用DNA双螺旋来组织记忆数据：

一条链是**过程态链** — 记录演变（first_seen_at / version / updated_at / decay_score）
另一条链是**关系态链** — 记录连接（分类归属 / 数据来源 / 记忆类型）
碱基对是记忆条目本身的内容

两条链缠绕在一起，就像生物DNA的双螺旋结构。过程态链告诉你"这条记忆怎么变的"，关系态链告诉你"这条记忆和什么关联"。两者结合，才是完整的记忆基因信息。

### SelfMind和其他方案的关系

| 层级 | 负责方 | 内容 |
|---|---|---|
| 高频内化层 | Hermes soul.md | 用户偏好、核心身份、常用工作模式——Agent"生来就知道" |
| 低频参考层 | Honcho Provider / SelfMind查询 | 历史对话上下文、存量记忆查询——Agent"需要时去查" |
| 过程追踪层 | SelfMind | 记忆的演变轨迹——"记忆怎么变的" |

SelfMind不和其他方案竞争，它补的是**最缺失的一层**——过程态。

## 五、过程态的价值

"记忆怎么变的"这件事，有什么用？

### 诊断用

像体检报告一样诊断Agent的记忆健康状况。哪些记忆强度高（常被使用）、哪些在衰退（长期未访问）、哪些有版本变化（内容在演化）。Agent自己感觉不到这些——它只知道自己"此刻有什么记忆"，不知道哪些记忆正在悄悄衰减。

### 优化用

衰减到阈值的记忆，要么强化（提醒Agent复习），要么归档（为新记忆腾空间）。就像人类大脑的遗忘机制——不是所有记忆都应该永远保留，健康遗忘是高效运作的前提。

### 回滚用

某个决策出了问题，可以追溯"当时Agent基于什么记忆做出这个判断"。是记忆过期了？还是理解错了？有了过程态数据，就能回放决策当时的记忆状态。

### 沉淀用

跨时间维度提炼高频模式。比如"这个月failure类记忆减少了30%，说明错误在收敛"——这是单条记忆看不到的宏观趋势。

### 反哺实时态

最核心的价值：过程态训练记忆，反哺实时态。

SelfMind分析发现某条记忆长期未用、衰减到阈值以下——告诉Agent"这条快忘了，要不要复习一下"。Agent把这条记忆重新加载到context里，刷新强度。

SelfMind发现某个行为模式反复出现——提炼成一条新的procedural记忆，写入高频区。

SelfMind发现某些记忆已经过时——建议从高频区移除，为新的记忆腾空间。

**这就是闭环：实时态产生行为 → SelfMind记录过程态 → 过程态训练记忆 → 反哺实时态 → Agent带着更新的记忆继续工作。**

## 六、从知识管理到知识工程

SelfMind的DNA测序仪定位，和商业银行正在推的"知识工程"方向完全一致。

传统知识管理（KM）把知识当成静态文档——写出来、存起来、需要时查一下。这对应的就是当前Agent记忆的"存储+查询"模式。

知识工程把知识当成活的系统——在使用中不断训练、演变、反哺。这对应的就是SelfMind的过程态追踪+反哺闭环。

从"存储知识"到"进化知识"，这是Agent记忆的范式转换。SelfMind是这个转换的工程实现。

每个Agent使用越久，SelfMind里积累的DNA数据就越丰富。当你需要创建一个新Agent时，可以"继承"成熟Agent的DNA数据。或者更有意思的——把几个Agent的优秀特征"杂交"：A的故障排查经验 + B的文档规范 + C的沟通风格，组合成新Agent的初始DNA。

这不是科幻，这是工程。DNA测序仪已经在这里了。

---

*SelfMind — Agent的DNA测序仪。不是记忆仓库，不是查询工具，是让Agent进化过程变得可追踪、可管理、可利用的工程系统。*
